# Contributing to First-PR

We welcome contributions to the First-PR project! By participating in this project, you agree to abide by our [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

There are many ways to contribute, from writing documentation to submitting bug fixes or new features.

### Reporting Bugs

If you find a bug, please open an issue on our [GitHub Issues page](https://github.com/your-username/First-PR/issues). When reporting a bug, please include:

*   A clear and concise description of the bug.
*   Steps to reproduce the behavior.
*   Expected behavior.
*   Screenshots or error messages if applicable.
*   Your operating system and browser information.

### Suggesting Enhancements

We love new ideas! If you have a suggestion for an enhancement or a new feature, please open an issue on our [GitHub Issues page](https://github.com/your-username/First-PR/issues). Describe your idea clearly and explain why you think it would be a valuable addition to the project.

### Code Contributions

1.  **Fork the repository:** Click the "Fork" button at the top right of the [First-PR GitHub page](https://github.com/your-username/First-PR).
2.  **Clone your fork:**
    ```bash
    git clone https://github.com/your-username/First-PR.git
    cd First-PR
    ```
3.  **Create a new branch:** Choose a descriptive name for your branch (e.g., `feature/add-dark-mode`, `bugfix/fix-login-error`).
    ```bash
    git checkout -b your-new-branch-name
    ```
4.  **Set up your development environment:** Follow the instructions in the main [README.md](README.md) file for setting up the backend and frontend.
5.  **Make your changes:** Write your code, tests, and update documentation as necessary.
6.  **Test your changes:** Ensure that your changes work as expected and don't introduce new bugs. Run existing tests and add new ones if appropriate.
7.  **Commit your changes:** Write clear, concise commit messages.
    ```bash
    git commit -m "feat: Add dark mode toggle"
    ```
8.  **Push to your fork:**
    ```bash
    git push origin your-new-branch-name
    ```
9.  **Open a Pull Request:** Go to the original [First-PR GitHub page](https://github.com/your-username/First-PR) and you should see a prompt to open a pull request from your new branch. Provide a clear title and description for your pull request, explaining the changes you've made and why they are beneficial.

## Code Style

*   **Python (Backend):** Adhere to PEP 8 guidelines. We use `black` for formatting and `flake8` for linting.
*   **JavaScript/React (Frontend):** Follow standard React best practices. We use `ESLint` for linting and `Prettier` for formatting.

## Review Process

Once you open a pull request, a maintainer will review your changes. They may provide feedback or request further modifications. Please be responsive to comments and willing to iterate on your changes.

Thank you for contributing to First-PR!
